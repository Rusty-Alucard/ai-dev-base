# Claude Code Usage Report

`bin/claude-usage.py` is a CLI that parses Claude Code transcript JSONL files and generates a weekly usage report covering skills, agents, MCP tools, token consumption, estimated cost, and Learnings tag frequency.

## Dependencies

```bash
pip install pyyaml matplotlib
```

## Basic Usage

```bash
python bin/claude-usage.py --range 7d --output logs/claude-usage/
```

## Common Options

| Option | Default | Description |
|--------|---------|-------------|
| `--range {7d,iso-week}` | `7d` | `7d` = last 7 days; `iso-week` = current ISO week (Mon–Sun) |
| `--project PATH` | — | Path to the Claude project JSONL directory (`~/.claude/projects/<slug>/`) |
| `--output DIR` | `logs/claude-usage/` | Output directory for the report and chart images |
| `--config PATH` | `config/claude-usage.yaml` | Config YAML path |
| `--period Nd` | — | Backward-compat alias for a custom day window (e.g. `--period 30d`) |

`--range` is the current flag; `--period` is preserved for older invocations. New code should use `--range`.

The project path can also be set via the `CLAUDE_PROJECT_PATH` environment variable or the `project_path` key in the config YAML. If none of these is provided, the script exits with an error.

## Config File

Copy the example config and fill in your project slug:

```bash
cp config/claude-usage.yaml.example config/claude-usage.yaml
```

Key settings:

- `project_path` — path to your `.claude/projects/<slug>/` directory
- `pricing` — USD per 1M tokens per model family (used for cost estimation)
- `ai_insight.enabled` — set `true` to append an AI-generated summary via the `claude` CLI
- `learnings.enabled` — set `true` to parse `## Learnings` sections from `.claude/agents/*.md` and include tag frequency in the report

## Output

The script writes to the `--output` directory:

```
logs/claude-usage/
  YYYY-MM-DD_weekly.md    # Markdown report (date is run date)
  images/
    skill_usage.png
    agent_spawn_frequency.png
    mcp_usage.png
    tool_type_usage.png
    cost_daily.png
    model_token_consumption.png
    daily_activity.png
```

> **Note:** `--output` is a **directory**, not a file path. The report filename is auto-generated as `YYYY-MM-DD_weekly.md`. Passing a `.md` file path to `--output` will create a subdirectory with that name and place the report inside it — not the behavior you want.

## Report Sections

1. **Skills** — invocation counts per skill name, activation rate
2. **Subagent** — agent spawn frequency by `subagent_type`
3. **MCP** — MCP tool usage by service
4. **Tool Type** — all tool call counts (MCP calls grouped)
5. **CTX / Cost** — daily cost estimate, `/clear` invocation count
6. **Sum** — totals and token share by model family
7. **Learnings** *(if enabled)* — tag frequency, promotion candidates (3-strike rule)
8. **AI Insight** *(if enabled)* — bullet-point summary generated by `claude -p`
